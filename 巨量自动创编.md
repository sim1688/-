### 服务重启提醒

`server.js` 修改后，浏览器不会自动使用新代码，必须重启本地 Node 服务。

如果页面没有看到接口变更，通常是旧进程仍在监听：

```powershell
netstat -ano | Select-String ':5173'
```

处理方式：

1. 关闭正在运行的 `start-server.bat` 黑色窗口。
2. 重新双击 `start-server.bat`。
3. 刷新 `http://localhost:5173/`。

已遇到案例：页面仍提示 `/open_api/2/agent/advertiser/select/` 权限不足，是因为旧服务未重启；新代码应优先调用 `/open_api/2/ebp/advertiser/list/`。

## 组织账号与广告主账号

### 重要结论

`/open_api/oauth2/advertiser/get/` 返回的是授权/管理账号列表，不一定是最终投放广告主列表。

当前验证到：

```text
1855464436371721 / 九四好玩-纳苓
```

这是组织维度账号，不是广告主投放账号。

### 组织下广告主列表

组织维度下的 300 多个广告主账号，需要用 EBP 工作台接口：

```text
GET /open_api/2/ebp/advertiser/list/
```

参数：

```text
enterprise_organization_id=1855464436371721
account_source=AD
page=1
page_size=100
```

已验证返回：

```text
total_number: 331
第一条：傲星-纳苓-330 / 1862696727384073
```

网页“更改”媒体账户弹窗已接入该接口，会展开组织下所有广告主账号。

### 代理商接口误区

曾尝试：

```text
/open_api/2/agent/advertiser/select/
```

返回：

```text
advertiser does no grant you /agent/advertiser/select/ permission
```

结论：当前场景不是代理商接口优先，而是组织/工作台 EBP 账号列表接口。

项目预算不能低于 300。

### 名称不能重复

若项目名称与已有项目重复，接口返回：



### 创建字段坑点

#### optimize_goal

查询返回中 `convert_id`、`asset_ids` 可能为 `null`。

不能带 `null` 进入创建请求，否则会报：

```text
optimize_goal.convert_id: Value is not nullable
optimize_goal.asset_ids: Value is not nullable
```

也不能强行给该小游戏场景传 `asset_ids`，否则报：

```text
该投放内容不支持选择事件资产
```

当前跑通做法：

```json
{
  "external_action": "AD_CONVERT_TYPE_GAME_ADDICTION",
  "external_actions": [25]
}
```

#### 时间格式

查询返回：

```text
2025-04-21 00:00:00
```

创建接口要求日期格式：

```text
YYYY-MM-DD
```

否则报：

```text
delivery_setting.start_time: Not a string of format: datetime=2006-01-02
```

#### 版位字段

查询返回：

```json
"inventory_type": ["INVENTORY_UNIVERSAL"]
```

创建接口不接受该枚举，报：

```text
delivery_range.inventory_type.0: value is not one of the allowed values
```

当前跑通做法：仅保留智能版位目录：

```json
{
  "inventory_catalog": "UNIVERSAL_SMART"
}
```

## 大整数 ID 精度问题

巨量 ID 通常超过 JavaScript 安全整数范围。

例如：

```text
7646712450741567507
```

如果用 `JSON.parse` 后再 `JSON.stringify`，可能会变成：

```text
7646712450741567000
```

因此：

- 读取展示时尽量保存原始字符串。
- 向巨量接口提交时，如果接口要求 integer，不能传字符串。
- 当前做法是构造 JSON 字符串时手动替换 ID 字面量，避免 JS 数字精度丢失。
